# 展会名片自动跟进流水线

**展会回来一堆名片，几十个客户要录入、要背调、要写开发信，一通忙活。**

这个工具把这件事交给 AI：把名片照片丢进文件夹，说一句"处理新名片"，十几分钟后你会得到一张 Excel 表——每个客户一行，名片信息、公司背调、写好的英文开发信全都填好了。你只需核对，然后复制粘贴发送。

不限行业。公司是做什么的、目标客户是谁，由你自己的知识库定义。

---

## 效果长这样

处理完的 Excel 表每行包含三段：

| 分区 | 内容 |
|---|---|
| ① 名片信息 | 姓名、职位、公司、邮箱、电话、网址、国家、名片原图、展会交谈要点 |
| ② 客户背调 | 公司简介、客户类型（零售商/品牌商/中间商/电商卖家…）、采购角色、联系人背景、与你业务的关联度+理由、信息来源网址 |
| ③ 开发信 | 英文邮件主题 + 正文，已结合展会交谈内容和背调结果个性化 |

---

## 你需要准备什么

1. **一个能读文件、能联网搜索的 AI 助手**。本工具的核心是一份写给 AI 看的说明书（`SKILL.md`），任何有文件访问和联网能力的 AI Agent 都能照着执行——例如 Claude（Cowork）、Codex、Kimi、WorkBuddy 等。把说明书交给你手头的 AI 即可（见下方"安装"）。
2. **一个文件夹**，放名片照片和产出的表格。
3. **公司资料**：公司介绍、产品资料、你们写得好的开发信（有几封拿到过回复的最好）。散乱没关系，AI 会帮你整理。

不需要写代码，不需要注册任何 API。（背调和建表用到联网与 Python，主流 AI Agent 都自带这些能力。）

## 安装

先把仓库拿到本地：

```bash
git clone https://github.com/xuduidui123/trade-show-lead-pipeline.git
```

（不用命令的话，在仓库网页点绿色 "Code" → "Download ZIP" 下载解压也一样。）

核心就是里面的 `trade-show-lead-pipeline/` 文件夹——它是一份纯文本说明书 `SKILL.md` 加两个脚本。让你的 AI 助手读到它即可，按你用的工具选一种：

- **Claude（Cowork / 桌面版）**：设置 → Capabilities → 添加 Skill，选中 `trade-show-lead-pipeline/` 文件夹；或把仓库里的 `trade-show-lead-pipeline.skill` 发给它，点 "Save skill"。
- **Codex / Kimi / WorkBuddy 等其他 AI Agent**：在你的工作文件夹里放上 `SKILL.md`，或对话开始时让 AI 先读这个文件，然后按 SKILL.md 里的触发词（如"处理新名片"）使用。不同工具的"加载说明书"方式略有差别，核心都是让 AI 能读到 `SKILL.md`。

> `.skill` 文件只是把 `trade-show-lead-pipeline/` 文件夹打包成的一个压缩文件，方便 Claude 一键安装；用其他工具时直接用文件夹里的 `SKILL.md` 即可，两者内容相同。

## 第一次使用（约 15 分钟）

**第 1 步：建工作文件夹**

新建一个文件夹（比如叫"客户开发工作流"），让你的 AI 助手访问这个文件夹，然后说：

> 初始化工作文件夹

AI 会自动建好目录结构、客户表模板和知识库占位文档。

**第 2 步：喂知识库**

把公司介绍、产品资料、优秀开发信统统丢进 `02-知识库/原始素材/`（Word、PDF、txt 都行，乱一点没关系），然后说：

> 整理知识库

AI 会读完所有素材，提炼成两份正式文档：`公司介绍.md`（你的卖点弹药库）和 `开发信范例.md`（写信的风格标准）。

打开这两份文档看一眼，尤其是"红线"部分——AI 会把素材里其他客户的销售数据之类的敏感信息标为红线，保证不写进开发信。

**第 3 步：试跑 3-5 张名片**

先别上几十张。放 3-5 张进 `01-名片照片/待处理/`，说：

> 处理新名片

跑完打开客户表，看开发信风格是否合意。不满意就直接说"语气再轻松点"、"开头别提公司规模"，满意后说"把这个标准更新进开发信范例"，以后就都按新标准写。

**这一步花的时间最值**——调好一次，后面每一封都受益。

## 日常使用（展会后）

1. 名片照片全部放进 `01-名片照片/待处理/`。
2. 在同一文件夹里的 `交谈要点采集.txt` 按名片序号记下现场聊了什么：

   ```
   1：DNS，想扩厨具品类，10月前定方案
   3：对柚木砧板感兴趣，有Amazon店铺WoodLuv，要电商包装报价
   ```

   也可以展会现场直接把要点手写在名片上再拍照，AI 一样能读。**这一步是开发信个性化的最大弹药，写得越具体，信越像人写的。**
3. **关掉客户表.xlsx**（开着会写入失败）。
4. 对你的 AI 助手说：`处理新名片`
5. 打开表格核对，复制开发信发送。

其他能说的话：`只做背调`、`只写开发信`、`重写第3行的开发信`、`整理知识库`。

## 发送前一定要核对

AI 会犯错，这几处必须人工过一遍：

- **邮箱和姓名拼写**——发错就是白费一个客户。
- 状态标了 ⚠️待人工核对 的行（照片糊了、字段缺失）。
- 标注"由邮箱域名推断"的网址。
- 背调写"信息有限"的行——这些客户的开发信是通用版，值得你手动加一句个性化内容。

工具的设计原则是**宁可留空也不编造**。如果某个字段是空的，那是它老实告诉你没查到，而不是漏了。

## 常见问题

**提示写入失败 / 文件被占用**
客户表还开着，关掉后说"继续"。

**某张名片识别得乱七八糟**
照片太糊。重拍一张放进待处理，说"重新处理这张名片"。

**开发信读起来像模板**
八成是知识库太单薄，或者没写交谈要点。补充这两处后重跑，效果差别很大。

**Excel 打开提示"内容有问题，是否恢复"**
表格结构被改坏了。跟 AI 说这个报错，它会重建一张干净的表并保留数据。

**背调查不到公司信息**
小公司没官网很正常，工具会如实标注并改用通用版开发信。这不是 bug。

## 关于隐私

名片含客户的个人信息（姓名、邮箱、电话）。使用时请注意：

- **客户表和名片照片不要传到公开仓库**，本仓库的 `.gitignore` 已经帮你挡了一层。
- 背调只使用公开可得的信息（官网、公司注册信息、LinkedIn 公开资料）。
- 你所在地区如果适用 GDPR 等法规，发送商业邮件前请自行确认合规要求。

## 想改造成自己的版本

Skill 本质上就是一份写给 AI 看的说明书，`SKILL.md` 是纯文本，直接改就行。常见改法：

- 想加字段（比如"报价状态""下次跟进日期"）→ 改 `scripts/build_table.py` 的字段定义
- 想让背调多查一项 → 改 SKILL.md 第二步的表格
- 想写多语言开发信 → 改第三步，加一句"按客户所在国语言撰写"
- 想改用飞书多维表格而非 Excel → 改 Excel 规范那一节

欢迎提 Issue 和 PR。
