# 展会客户开发信自动化流水线

把展会拿到的名片照片，自动变成"名片信息+客户背调+个性化英文开发信"三段俱全的Excel客户表。业务员只需核对和复制粘贴发送。

本Skill不预设行业——公司主营什么、目标客户是谁，全部由用户的知识库（`02-知识库/公司介绍.md`）定义。判断"业务关联度"时以知识库描述的品类为准。

## 工作文件夹约定

在用户的工作文件夹内使用以下结构。**首次使用时若结构不存在，运行 `scripts/init_workspace.py <文件夹路径>` 一键创建**（会生成文件夹、客户表模板、知识库占位文档和交谈要点采集模板）：

```
├── 01-名片照片/待处理/    ← 名片照片 + 交谈要点采集文档放这里
├── 01-名片照片/已处理/    ← 处理完自动移入，文件名前加行号
├── 02-知识库/
│   ├── 公司介绍.md        ← 公司卖点、产品线、客户名单、红线
│   ├── 开发信范例.md      ← 风格标准+范例，写信时优先级最高
│   └── 原始素材/          ← 用户丢散乱资料，说"整理知识库"时读取
└── 客户表.xlsx            ← 最终产出
```

知识库还是占位版时：提醒用户放素材，同时先用通用标准干活，不要卡住流程——先跑通比先完美重要。

## 触发词

- "处理新名片" → 完整跑三步
- "只做背调" / "只写开发信" → 单独跑对应步骤（按状态列筛选行）
- "整理知识库" → 读 `02-知识库/原始素材/` 全部文件，提炼更新公司介绍.md 和开发信范例.md。注意：素材中若含具体客户的销售数据/合作细节，写入"红线"清单而非卖点——开发信里泄露其他客户数据会严重损害信任。
- "重写第X行的开发信" → 重读该行（用户可能补充了展会备注）后重新生成

## 第一步：名片提取 → ①区

1. 逐张识别 `01-名片照片/待处理/` 里的图片，提取：姓名、职位、公司名、邮箱、电话、网址、国家/地址。
2. **采集交谈要点**（这是开发信个性化的最大弹药，务必认真）：
   - 读取待处理文件夹里名为 **`交谈要点采集`** 的文档（.docx/.txt），业务员按名片序号逐条记录（如"3：对柚木砧板感兴趣，要电商包装报价"）。按序号匹配写入对应行"展会备注"。
   - 名片上的手写字也一并采集，与文档内容合并去重。
   - 文档里的序号对不上任何照片时，在汇总时提示用户。
3. 写入客户表①区，一张名片一行，状态=⬜待处理，录入日期=当天；名片原图缩略图嵌入"名片照片"列。
4. 诚实原则：看不清/缺失的字段留空并把状态标"⚠️待人工核对"——**绝不猜测编造**，一个错邮箱就是一次浪费的商机。网址未印但邮箱是企业域名时可推断，须注明"由邮箱域名推断"。同公司第二张名片正常录入但标"（同公司重复）"。
5. 处理完的照片移入 `已处理/`，文件名前加行号（如 `03_原名.jpg`）。

## 第二步：客户背调 → ②区

对每个 ⬜待处理 行，按公司名+网址联网调查：

| 字段 | 标准 |
|---|---|
| 公司简介 | 主营、成立时间、规模、所在地，2-3句 |
| 客户类型 | 超市/连锁零售、品牌商、进口商/分销商、亚马逊/电商卖家、中间商/采购办、终端用户、其他 |
| 采购角色判断 | 结合职位+公司规模：决策人/影响者/执行者/无法判断 |
| 联系人背景 | LinkedIn等公开信息；查不到写"未查到公开信息" |
| 业务关联度 | 高/中/低，判断轴=对方品类与我司品类重合度+采购模式匹配度（我司品类以知识库公司介绍.md为准） |
| 关联度理由 | 1-2句判断依据 |
| 信息来源 | 来源名称**+具体网址**（每条一行），方便人工核查 |

查不到的如实写"信息有限"，照常进第三步用通用版。有合规风险的市场（如受制裁地区）在理由里提示。完成后状态改 🔍已背调。

## 第三步：生成开发信 → ③区

1. 先读知识库两个文档，开发信范例.md 的风格标准优先级最高。
2. 每个 🔍已背调 行生成英文开发信1封（主题+正文）。
3. 个性化优先级：**展会备注的交谈内容 > 背调发现的业务特点 > 国家/渠道通用痛点 > 通用版**。至少引用1个该客户的具体信息——群发感是开发信被删的第一原因。
4. 遵守知识库红线；不承诺价格；不用垃圾邮件词（free, best price, 100%）；首封不塞多附件。
5. 完成后状态改 ✅已生成，特殊情况写入"生成备注"。

## Excel规范（重要，违反会损坏文件）

- 表结构、配色、列宽见 `scripts/build_table.py`（22列4分组：管理A-C灰/①名片D-L蓝/②背调M-S绿/③开发信T-V黄，第1行组名合并居中，第2行字段名，冻结D3）。
- 名片缩略图：宽度≤200px（2.1英寸），锚定到该行L列——过大会遮挡相邻单元格。
- **改表格结构（增删列）绝不用 insert_cols 直接插列**——会打断合并单元格，Excel打开报"内容有问题"。正确做法：读出全部数据→重建干净工作簿→重写表头合并区→写回数据和图片→验证后覆盖。
- 写表前先读表：已有数据行不覆盖，新数据追加到末尾。
- 写入失败提示文件被占用 → 请用户关闭Excel后说"继续"。

## 每批收尾

向用户汇总：处理了几张、几行⚠️待人工核对、几行信息有限、哪些行关联度高值得优先跟进。提醒：发送前人工核对邮箱和姓名拼写；推断的网址需确认。
