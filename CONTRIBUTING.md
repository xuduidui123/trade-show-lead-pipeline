# 参与改进

欢迎任何形式的贡献，尤其欢迎来自一线业务员的真实反馈——你们知道什么样的开发信才有回复。

## 提 Issue 时

如果是"效果不好"类的问题，请尽量说明：

- 你的行业和产品类型
- 具体哪一步出了问题（名片识别 / 背调 / 开发信）
- 期望的结果 vs 实际结果

**提交前请务必删掉真实客户信息**——姓名、邮箱、电话、公司名。用 `John Smith / ABC Ltd.` 之类的假名替代即可。

## 提 PR 时

本项目的核心是 `trade-show-lead-pipeline/SKILL.md`，它是一份写给 AI 看的说明书，改动就是改文字。请注意：

- **解释"为什么"，而不是堆砌命令。** 与其写"必须填写信息来源"，不如写"信息来源要附网址，方便业务员人工核查"——AI 理解了意图，在你没预料到的场景里也能做对。
- **保持行业中立。** 具体行业的知识应该放进用户自己的知识库，不要写死在 Skill 里。
- **改动请先自己跑一遍**，用几张假名片验证效果。

改 `scripts/` 里的 Python 脚本时，请保证 `init_workspace.py` 在空文件夹里能一次跑通。

## 项目结构

```
├── README.md                       使用说明（面向业务员，非技术）
├── trade-show-lead-pipeline/
│   ├── SKILL.md                    核心：给 AI 的作业手册
│   └── scripts/
│       ├── init_workspace.py       一键初始化工作文件夹
│       └── build_table.py          生成客户表模板
└── .gitignore                      挡住客户数据，别删
```
